# 🎮 Tic-Tac-Toe Backend API

REST-сервис для игры в «Крестики-нолики» с поддержкой игры против компьютера (алгоритм Minimax) и PvP-режима, авторизацией через JWT, историей игр и таблицей лидеров. Проект последовательно развивался от in-memory хранения до полноценной работы с PostgreSQL и защищённой сессией на основе токенов.

## 🛠 Технологический стек
- **Язык:** Java 18
- **Фреймворк:** Spring Boot (Web, Security, Data JPA)
- **Сборка:** Gradle (Kotlin DSL)
- **База данных:** PostgreSQL + Hibernate / JPA
- **Безопасность:** JWT (access + refresh tokens), кастомные фильтры, ролевая модель
- **Архитектура:** Слоистая (`web`, `domain`, `datasource`, `di`), строгое разделение DTO/Entity/Domain-моделей

## ✨ Основной функционал
- 👤 **Авторизация:** регистрация, вход, выдача `accessToken` + `refreshToken`, механизм обновления сессий.
- 🤖 **Игра против ИИ:** сервер рассчитывает оптимальный ход компьютера с помощью алгоритма Minimax.
- 👥 **PvP-режим:** создание игр, ожидание второго игрока, присоединение, валидация очередности ходов.
- 📜 **История игр:** получение всех завершённых партий текущего пользователя.
- 🏆 **Лидерборд:** расчёт винрейта (победы / (проигрыши + ничьи)), сортировка, вывод топ-N игроков.
- 🔒 **Безопасность:** кастомный `AuthFilter`, валидация JWT, проверка ролей, защита эндпоинтов через `SecurityFilterChain`.

## 📐 Архитектура проекта
Проект разделён на изолированные слои с чёткими границами ответственности:

| Слой | Назначение |
|------|------------|
| `web` | Контроллеры, DTO-модели запросов/ответов, мапперы `web ↔ domain` |
| `domain` | Бизнес-логика, интерфейсы сервисов, доменные модели игры и пользователя |
| `datasource` | JPA-сущности, `CrudRepository`-репозитории, мапперы `domain ↔ datasource`, операции с БД |
| `di` | Spring-конфигурации, бины безопасности, `JwtProvider`, граф зависимостей |

> Все интерфейсы, реализации и модели вынесены в отдельные файлы для соблюдения принципа единственной ответственности.

## 🌐 Основные эндпоинты API
> Базовый путь: `http://localhost:8080`  
> Для авторизованных методов требуется заголовок: `Authorization: Bearer <accessToken>`

### 🔐 Авторизация
| Метод  | Путь | Описание |
|--------|------|----------|
| `POST` | `/auth/register` | Регистрация нового пользователя |
| `POST` | `/auth/login` | Авторизация (возвращает пару токенов) |
| `GET`  | `/auth/refresh/access` | Обновление `accessToken` |
| `GET` | `/auth/refresh/refresh` | Обновление `refreshToken` |
| `GET`  | `/auth/me` | Получение профиля из текущего токена |

### 🎲 Игры
| Метод | Путь               | Описание |
|-------|--------------------|----------|
| `POST` | `/game/start`      | Создание новой игры (PvE или PvP) |
| `GET`  | `/game/available`  | Список доступных игр для присоединения |
| `POST` | `/game/join{uuid}` | Присоединение к существующей игре |
| `POST` | `/game/{uuid}`    | Совершение хода (валидация + ход ИИ или противника) |
| `GET`  | `/game/{uuid}`    | Текущее состояние игры |

### 📈 Статистика
| Метод | Путь                      | Описание |
|-------|---------------------------|----------|
| `GET`  | `/game/my-games`          | Завершённые игры текущего пользователя |
| `GET`  | `/game/leaderboard?top=N` | Топ-N лучших игроков по винрейту |

## ⚙️ Установка и запуск
Запуск доступен напрямую из IDE. API станет доступен по адресу http://localhost:8080.
